# Agent Guidelines

Instructions for LLMs working on this chezmoi dotfiles repository.

## Core workflow

This is a **chezmoi** repo. The source of truth is `~/.local/share/chezmoi/` (this directory). Files are deployed to `~/` via `chezmoi apply`.

- **Edit files here** (the chezmoi source directory), not in `~/.config/` directly.
- After editing, run `chezmoi apply` or `chezmoi apply ~/.config/<target>` to deploy changes.
- To preview what would change: `chezmoi diff`.
- To see what chezmoi manages: `chezmoi managed`.

### Chezmoi naming rules

| Source prefix/suffix | Effect |
|---|---|
| `dot_` | Becomes `.` in target path |
| `private_` | Deployed with 0600 permissions |
| `.tmpl` | Go-templated before deployment |
| `executable_` | Deployed with executable bit |
| `run_once_` | Script runs once (first apply) |
| `run_onchange_` | Script re-runs when its content hash changes |
| `run_after_` | Script runs after every apply |

### Template variables

Available in `.tmpl` files — defined in `.chezmoi.toml.tmpl`:

- `{{ .machine_type }}` — `"work"`, `"dev"`, or `"casual"`
- `{{ .brew_prefix }}` — `/opt/homebrew` (arm64) or `/usr/local` (x86)
- `{{ .chezmoi.hostname }}` — machine hostname
- `{{ .cert_path }}`, `{{ .jira_api_token }}` — secrets (never commit values)
- `{{ .tuna_activate_keycode }}`, etc. — Tuna hotkey codes

## Service management

### Sketchybar

Sketchybar runs as a **brew service**, not a shell process.

```sh
# Correct
brew services start sketchybar
brew services restart sketchybar

# Wrong — ties process to terminal session
sketchybar                    # DO NOT run directly
exec-and-forget sketchybar    # only as AeroSpace fallback
```

To reload config without restarting the service:
```sh
sketchybar --reload
```

Sketchybar uses **SbarLua** (Lua bindings). The config entry point is `dot_config/sketchybar/sketchybarrc` (a Lua script). Colors are generated by the theme pipeline — `colors.lua` and `colors.sh` are in `.chezmoiignore` because they're regenerated at runtime.

### AeroSpace

AeroSpace is the tiling window manager. Its config is templated (`aerospace.toml.tmpl`) to handle:
- Monitor-specific gap calculations (notch vs external displays)
- Machine-specific behavior (hostname checks)

After editing the template, apply and reload:
```sh
chezmoi apply ~/.config/aerospace/aerospace.toml
# AeroSpace picks up changes automatically, or use service mode: alt-shift-s then esc
```

## Key architectural decisions

### Workspace model

16 workspaces in 4 groups: A1-A4, B1-B4 (primary monitor), C1-C4, D1-D4 (secondary monitor). The `workspace-to-monitor-force-assignment` section in `aerospace.toml.tmpl` handles fallback to main display when undocked. Sketchybar's `spaces.lua` hides empty secondary workspaces on single-monitor setups.

### Theme pipeline

Colors flow from Emacs theme palettes through a multi-stage pipeline:
1. `dot_config/elvish/lib/dotfiles/palette.elv` — syncs theme repos
2. `dot_config/dotfiles/parse-palette.bb` — Babashka script extracts palettes to JSON
3. `dot_config/elvish/lib/dotfiles/theme.elv` — applies theme across apps
4. Generated files (`colors.lua`, `colors.sh`, wallpapers) are **not checked in** — they're runtime artifacts

When modifying theme-related code, don't edit generated `colors.*` files. Edit the generators in `elvish/lib/dotfiles/` or `parse-palette.bb`.

### Fonts

The repo uses Monaspace font family (Neon, Argon variants) for UI elements and Aporetic Sans for labels. These are installed via Homebrew casks in the Brewfile.

### Multi-machine support

Conditional behavior is driven by `{{ .machine_type }}`:
- **Brewfile**: gates packages/casks per machine type
- **AeroSpace**: notch-aware gap calculation checks hostname
- **Fish/Elvish**: cert paths and tokens for work machines

When adding machine-specific behavior, use template conditionals rather than runtime checks where possible.

## File relationships

- `run_onchange_01-install-brew-packages.sh.tmpl` watches `dot_Brewfile.tmpl` via `{{ include "dot_Brewfile.tmpl" | sha256sum }}`
- `run_onchange_02-install-mise-tools.sh.tmpl` watches `dot_config/mise/config.toml`
- `run_onchange_03-install-fisher-plugins.sh.tmpl` watches `dot_config/fish/fish_plugins`
- `run_after_apply-theme.sh.tmpl` runs after every apply to sync the theme database

Changing a watched file triggers its corresponding `run_onchange` script on next `chezmoi apply`.

## Common tasks

**Add a new Homebrew package**: Edit `dot_Brewfile.tmpl`, wrap in machine_type conditional if needed.

**Add a new tool version**: Edit `dot_config/mise/config.toml`.

**Modify sketchybar**: Edit files in `dot_config/sketchybar/`, then `chezmoi apply && sketchybar --reload`.

**Modify AeroSpace keybindings**: Edit `dot_config/aerospace/aerospace.toml.tmpl`, then `chezmoi apply`.

**Add a new Ghostty theme**: Add theme file to `dot_config/ghostty/themes/`, reference in theme pipeline if it's a Prot palette.

**Test template rendering**: `chezmoi execute-template < file.tmpl` or `chezmoi cat ~/.config/path/to/file`.
