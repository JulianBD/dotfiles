* About

This is my literate Emacs configuration, designed with ADHD-friendly workflows, comprehensive note-taking, and enhanced discoverability. The configuration prioritizes frictionless interaction and progressive complexity, making it ideal for Emacs newcomers while remaining powerful for advanced users.

** Quick Start

1. Place a minimal init.el in your ~/.emacs.d directory:
   #+begin_src emacs-lisp :tangle no
   ;; Initialize package system
   (require 'package)
   (add-to-list 'package-archives '("melpa" . "https://melpa.org/packages/") t)
   (package-initialize)

   ;; Install use-package if needed
   (unless (package-installed-p 'use-package)
     (package-refresh-contents)
     (package-install 'use-package))

   ;; Install org if needed
   (use-package org :ensure t)

   ;; Load the literate config
   (org-babel-load-file
    (expand-file-name "config.org" user-emacs-directory))
   #+end_src

2. Place config.org in your ~/.emacs.d directory
3. Start Emacs - it will:
   - Install required packages
   - Tangle the org file into individual .el files
   - Load the configuration

** Configuration Structure

This configuration is organized as a literate document using Org Babel to tangle code blocks into the appropriate Emacs Lisp files. The modular structure allows for targeted testing and maintains clean separation of concerns.

** Design Philosophy

*Ergonomics First*: Every feature is designed to reduce friction and cognitive load, supporting habit formation over memorization.

*Discoverability Over Efficiency*: Extensive use of which-key and visual feedback helps users discover features naturally rather than requiring extensive documentation study.

*Progressive Complexity*: Simple defaults that can grow more sophisticated as users become comfortable with the system.

*ADHD-Friendly Workflows*: Minimal decision-making during capture, consistent interfaces, and temporal organization over complex categorization.

** Startup performance optimization

During Emacs startup, garbage collection can cause noticeable pauses. By temporarily increasing the threshold to 100MB, we reduce these interruptions. The original value is saved so it can be restored after initialization completes.

#+begin_src emacs-lisp :tangle init.el
(defvar jd--initial-gc-threshold gc-cons-threshold
  "Initial value of `gc-cons-threshold' at startup.")
(setq gc-cons-threshold 100000000)
#+end_src

** Load path configuration

All custom modules live in the =lisp/= subdirectory. Adding this to the load path allows =require= statements to find our custom modules.

#+begin_src emacs-lisp :tangle init.el
(add-to-list 'load-path (expand-file-name "lisp"
                                         (file-name-directory load-file-name)))
#+end_src

** Package management setup

The package system needs configuration before any modules try to load packages. We add MELPA for the latest packages and GNU ELPA for stable ones, then ensure =use-package= is available as our primary package management tool.

#+begin_src emacs-lisp :tangle init.el
(require 'package)
(add-to-list 'package-archives '("melpa" . "https://melpa.org/packages/") t)
(add-to-list 'package-archives '("gnu" . "https://elpa.gnu.org/packages/") t)
(package-initialize)

(unless (package-installed-p 'use-package)
  (package-refresh-contents)
  (package-install 'use-package))

(eval-when-compile
  (require 'use-package))
(require 'use-package-ensure)

(setq use-package-always-ensure nil)
#+end_src

This helper function checks whether packages need to be installed by looking for a marker file. This prevents unnecessary network calls on every startup while ensuring packages are available when needed.

#+begin_src emacs-lisp :tangle init.el
(defun jd--package-install-needed-p ()
  "Return t if we need to install packages (first run)."
  (not (file-exists-p (expand-file-name "package-installed" user-emacs-directory))))
#+end_src

** Centralized backup system

Instead of cluttering the file system with backup files alongside originals, this system creates a mirror directory structure within =~/.emacs.d/backups/=. This keeps working directories clean while preserving the ability to recover from accidental changes.

#+begin_src emacs-lisp :tangle lisp/jd-base.el
(defun jd--backup-file-name (fpath)
  "Return a new backup path for FPATH.
Creates necessary directories if they don't exist."
  (let* ((backup-root-dir (expand-file-name "backups/" user-emacs-directory))
         (file-path (replace-regexp-in-string "[A-Za-z]:" "" fpath))
         (backup-file-path (replace-regexp-in-string "//" "/"
                                                   (concat backup-root-dir file-path "~"))))
    (make-directory (file-name-directory backup-file-path) t)
    backup-file-path))

(setq make-backup-file-name-function 'jd--backup-file-name)
#+end_src

(defun jd-next-user-buffer ()
  "Switch to the next user buffer (excludes system buffers)."
  (interactive)
  (let ((start-buffer (current-buffer)))
    (next-buffer)
    (while (and (string-match-p "^\\*" (buffer-name))
                (not (eq (current-buffer) start-buffer)))

(defun jd-themes-load-theme (theme)
  "Load the specified THEME.

  (interactive
   (list (intern (completing-read "Select theme: " 
                                  (mapcar #'symbol-name (jd-themes-list-all))))))

  (let ((collection (jd-themes-get-collection theme)))

     ((eq collection 'modus) (jd-themes-load-modus))
     ((eq collection 'ef) (jd-themes-load-ef))
     ((eq collection 'doric) (jd-themes-load-doric))

(defun jd-themes-toggle-dark-light ()
  "Toggle between dark and light variants within the same theme collection."
  (interactive)
  (let* ((current-theme (car custom-enabled-themes))
         (collection (jd-themes-get-collection current-theme))
         (themes (cdr (assq collection jd-themes--collection-map)))
         ;; Determine if current theme is dark
         (is-dark (cond
                   ;; Modus themes
                   ((eq collection 'modus)
                    (string-match-p "vivendi" (symbol-name current-theme)))
                   
                   ;; EF themes - use the comprehensive list of dark themes
                   ((eq collection 'ef)
                    (member current-theme '(ef-autumn ef-bio ef-blue ef-cyprus ef-dark
                                           ef-deuteranopia-dark ef-duo-dark ef-elea-dark
                                           ef-frost ef-kassio ef-maris-dark ef-melissa-dark
                                           ef-night ef-symbiosis ef-trio-dark ef-tritanopia-dark
                                           ef-winter)))
                   
                   ;; Doric themes
                   ((eq collection 'doric)
                    (not (eq current-theme 'doric-light)))
                   
                   ;; Default fallback for other themes
                   (t (string-match-p "dark\\|night" (symbol-name current-theme)))))
         
         ;; Filter themes to get opposite variants
         (opposite-variants 
          (cond
           ;; For Modus themes, simply swap between operandi and vivendi variants
           ((eq collection 'modus)
            (if is-dark
                (seq-filter (lambda (theme) (string-match-p "operandi" (symbol-name theme))) themes)
              (seq-filter (lambda (theme) (string-match-p "vivendi" (symbol-name theme))) themes)))
           
           ;; For EF themes, use the known light/dark theme lists
           ((eq collection 'ef)
            (if is-dark
                (seq-filter (lambda (theme) 
                             (not (member theme '(ef-autumn ef-bio ef-blue ef-cyprus ef-dark
                                                ef-deuteranopia-dark ef-duo-dark ef-elea-dark
                                                ef-frost ef-kassio ef-maris-dark ef-melissa-dark
                                                ef-night ef-symbiosis ef-trio-dark ef-tritanopia-dark
                                                ef-winter))))
                           themes)
              (seq-filter (lambda (theme)
                           (member theme '(ef-autumn ef-bio ef-blue ef-cyprus ef-dark
                                          ef-deuteranopia-dark ef-duo-dark ef-elea-dark
                                          ef-frost ef-kassio ef-maris-dark ef-melissa-dark
                                          ef-night ef-symbiosis ef-trio-dark ef-tritanopia-dark
                                          ef-winter)))
                         themes)))
           
           ;; For Doric themes
           ((eq collection 'doric)
            (if is-dark
                '(doric-light)  ;; Always switch to light when in any dark doric theme
              '(doric-dark)))   ;; Always switch to dark when in light doric theme
           
           ;; Default case for other theme collections
           (t (seq-filter
               (lambda (theme)
                 (if is-dark
                     (not (string-match-p "dark\\|night" (symbol-name theme)))
                   (string-match-p "dark\\|night" (symbol-name theme))))
               themes)))))
    
    (if opposite-variants
        (jd-themes-load-theme (car opposite-variants))
      (message "No appropriate light/dark variant found in the %s collection" collection))))

         (week-ago (time-subtract today (days-to-time 7)))
         (month-ago (time-subtract today (days-to-time 30)))
         (today-str (format-time-string "%Y-%m-%d" today))
         (week-ago-str (format-time-string "%Y-%m-%d" week-ago))
         (month-ago-str (format-time-string "%Y-%m-%d" month-ago))

                 (concat "_" (regexp-quote jd-journal-keyword))))

         (month-count 0)

        (when (and (string>= file-date month-ago-str)

          (setq month-count (1+ month-count)))))

   ((eq major-mode 'org-mode)
    '(:h1 "* " :h2 "** " :h3 "*** " :bold "*" :bold-end "*"
      :table-row "| " :table-sep " | " :table-end " |"))
   ((or (eq major-mode 'markdown-mode) (eq major-mode 'gfm-mode))
    '(:h1 "# " :h2 "## " :h3 "### " :bold "**" :bold-end "**"
      :table-row "| " :table-sep " | " :table-end " |"))
   (t ; text-mode or others
    '(:h1 "" :h2 "" :h3 "" :bold "" :bold-end ""
      :table-row "" :table-sep ": " :table-end ""))))

(defun jd-journal-weekly-ai-review ()
  "Prepare a weekly review for AI analysis."
  (interactive)
  (let* ((today (current-time))
         (week-ago (time-subtract today (days-to-time 7)))
         (today-str (format-time-string "%Y-%m-%d" today))
         (week-ago-str (format-time-string "%Y-%m-%d" week-ago))
         (buffer (get-buffer-create "*Weekly Journal Review*"))
         (files (denote-directory-files
                 (concat "_" jd-journal-keyword))))
    (with-current-buffer buffer
      (erase-buffer)
      (insert (format "# Weekly Journal Review: %s to %s\n\n"
                      week-ago-str today-str))
      ;; For each journal file in the past week
      (dolist (file files)
        (let ((file-date (format-time-string "%Y-%m-%d" 
                          (date-to-time (denote-extract-id-from-file-name file)))))
          (when (and (string>= file-date week-ago-str)
                     (string<= file-date today-str))
            (insert (format "## %s\n\n" file-date))
            (insert-file-contents file)
            (goto-char (point-max))
            (insert "\n\n"))))
      (goto-char (point-min))
      (display-buffer buffer))))
